# 查询模式与响应方式

## 查询分类

### A. 统计类查询

需要聚合计算，返回统计数据。

**模式**:
- `今年/去年/本月/上周 加了多少天班？`
- `一共多少通宵？`
- `花了多少钱？`
- `跑了多少公里？`
- `读了多少本书？`
- `写了多少笔记？`

**处理流程**:
1. 确定时间范围
2. 搜索对应目录的所有文件
3. 聚合相关字段
4. 计算统计值
5. 生成报告格式输出

**输出格式**:
```markdown
# {统计类型} — {时间范围}

## 统计结果
- **总计**: {数字} {单位}
- **日均**: {数字} {单位}
- **最高**: {数字} ({日期})

## 明细
| 日期 | 事项 | 数值 | 备注 |
|------|------|------|------|

## 趋势分析
{{AI分析内容}}
```

---

### B. 回顾类查询

需要按时间线组织，返回时间序列信息。

**模式**:
- `国庆7天我都去了哪里？`
- `上周做了什么？`
- `这一个月的生活总结`
- `去年生日怎么过的？`
- `第一次学车的经历`

**处理流程**:
1. 确定时间范围
2. 搜索 daily/trips/moments
3. 按时间排序
4. 提取关键事件
5. 组织成叙事格式

**输出格式**:
```markdown
# {回顾主题} — {时间范围}

## 时间线

### Day 1 — {日期}
- 事件概述
- 详情

### Day 2 — {日期}
...

## 总体回顾
{{AI总结}}
```

---

### C. 检索类查询

基于关键词，返回相关内容。

**模式**:
- `我记过哪些 React 的知识点？`
- `关于 Webpack 的笔记有哪些？`
- `学过哪些设计模式？`
- `去过哪些地方？`

**处理流程**:
1. 提取关键词
2. 搜索全库
3. 过滤相关笔记
4. 按相关性排序
5. 汇总输出

**输出格式**:
```markdown
# 关于 "{关键词}" 的记录

共找到 {N} 条相关记录:

## 技术笔记
- [[笔记1]] — {摘要}
- [[笔记2]] — {摘要}

## 日常记录
- [[日记1]] ({日期}) — {相关片段}
...

## 相关资源
- [外部链接](URL)
```

---

### D. 事实问答

直接回答具体问题。

**模式**:
- `去年我生日那天天气怎么样？`
- `第一次去日本是什么时候？`
- `我的年终奖发了多少？`
- `当时为什么做那个决定？`

**处理流程**:
1. 解析问题类型
2. 定位相关文件
3. 提取答案
4. 组织回答
5. 提供来源

**输出格式**:
```markdown
# 问题: {问题原文}

## 答案
{直接回答}

## 来源
- [[相关笔记]] ({日期})
```

---

### E. 关联分析

发现信息间的联系。

**模式**:
- `这个项目和之前那个有什么联系？`
- `我和老婆去过哪些相同的地方？`
- `这段时间状态不好的原因是什么？`
- `我的技术成长路径是什么？`

**处理流程**:
1. 提取实体（人/事/地点）
2. 搜索相关记录
3. 分析关联模式
4. 生成洞察

**输出格式**:
```markdown
# 关联分析: {主题}

## 发现
{AI发现的关联}

## 关联图谱
- {实体A} ↔ {实体B}: {关系}
- {实体C} ↔ {实体D}: {关系}

## 洞察
{基于关联的洞察}
```

### F. 生成报告

结构化生成特定类型的报告。

**报告类型**:

| 类型 | 命令示例 | 数据来源 |
|------|---------|---------|
| work-summary | 帮我做本月工作回顾 | work-logs, daily |
| travel-summary | 生成年度旅行报告 | trips, daily |
| health-summary | 健康状况分析 | health, daily |
| habit-summary | 习惯养成报告 | habits |
| mood-summary | 情绪变化分析 | moods |
| monthly-review | 月度复盘 | 全库 |

**通用格式**:
```markdown
# {报告类型} — {时间范围}

## 概览
- {关键指标1}: {数值}
- {关键指标2}: {数值}
- {关键指标3}: {数值}

## 数据详情
{详细数据表或列表}

## {类型化分析}

### {子标题1}
{分析内容}

### {子标题2}
{分析内容}

## {AI建议/洞察}
{基于数据的建议}
```

## 查询意图识别

| 意图 | 关键词 | 处理方式 |
|------|--------|---------|
| 统计 | 多少、几次、几天、多少钱 | 聚合计算 |
| 回顾 | 做了什么、去哪了、怎么过的 | 时间线组织 |
| 检索 | 记过、关于、有没有 | 关键词搜索 |
| 问答 | 是谁、什么时候、多少 | 直接提取 |
| 分析 | 为什么、什么原因、有什么关系 | 关联分析 |
| 生成 | 总结、报告、回顾 | 结构化报告 |

## 利用双链召回

利用 `backlinks` 字段快速定位相关信息：

**场景**: 查询 "小王是谁？"
1. 搜索 `3. Resources/people/小王.md`
2. 查看 `backlinks` 字段 → 获取所有相关日记、项目
3. 聚合信息生成回答

**场景**: 查询 "React 性能优化我记过哪些？"
1. 搜索 `3. Resources/tech/react性能优化.md`
2. 查看 `backlinks` 字段 → 获取所有提到过的日记、讨论
3. 结合 tech 笔记内容组织回答
