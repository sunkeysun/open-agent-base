# 主动关联规则

## 关联触发条件

在处理任何输入时，自动检查以下关联类型：

---

## 1. 时间关联

**触发条件**: 输入中包含时间信息

**检查项**:
- 是否与同一天的其他记录关联？
- 是否与去年同期事件关联？
- 是否形成周期性模式？（如每周一固定开会）

**关联方式**:
```markdown
## 时间关联
- 同日记录: [[4. Records/daily/YYYY-MM-DD]]
- 去年同期: [[4. Records/daily/YYYY-MM-DD]] (1年前)
- 周期模式: 发现这是你第 {N} 次 {事件}
```

---

## 2. 人物关联

**触发条件**: 输入中提到人名或人称代词

**检查项**:
- 之前是否与此人有过互动？
- 此人在哪些记录中出现过？
- 是否应该提醒用户关于此人的待办？

**关联方式**:
```markdown
## 人物关联
- [[姓名]]: 上次见面是 {日期} ({地点})
- 共同出现 {N} 次
- 关系: {朋友/同事/家人}
```

---

## 3. 地点关联

**触发条件**: 输入中提到地点

**检查项**:
- 之前是否去过此地？
- 这是新地点还是旧地重游？
- 与此地相关的其他记忆？

**关联方式**:
```markdown
## 地点关联
- [[地点]]: 上次访问是 {日期}
- 共去过 {N} 次
- 相关记忆: [[日记1]], [[日记2]]
```

---

## 4. 项目关联

**触发条件**: 输入中提到项目名或工作内容

**检查项**:
- 是否与已有的项目笔记关联？
- 是否应该更新项目进度？
- 是否有相关技术知识可以链接？

**关联方式**:
```markdown
## 项目关联
- 相关项目: [[1. Projects/work/项目名]]
- 项目阶段: {阶段}
- 相关技术: [[tech/xxx]]
```

---

## 5. 知识关联

**触发条件**: 输入中涉及技术/概念

**检查项**:
- 之前是否有过相关笔记？
- 是否有读书时学到的相关知识？
- 是否可以补充到已有知识网络？

**关联方式**:
```markdown
## 知识关联
- 相关笔记: [[3. Resources/tech/xxx]]
- 相关书籍: [[3. Resources/reading/书名]]
- 补充到: [[已有笔记]] (作为关联阅读)
```

---

## 6. 情绪关联

**触发条件**: 输入中包含情绪词

**检查项**:
- 最近情绪趋势如何？
- 是否有特定模式？（如周末心情好）
- 是否需要提醒用户关注情绪？

**关联方式**:
```markdown
## 情绪关联
- 近期心情: 平均 {X}/5
- 情绪趋势: {上升/下降/平稳}
- 相关因素: {可能的关联因素}
```

---

## 7. 习惯关联

**触发条件**: 输入中涉及习惯行为

**检查项**:
- 是否与习惯追踪相关？
- 是否应该打卡记录？
- 连续记录情况如何？

**关联方式**:
```markdown
## 习惯关联
- 习惯打卡: [[4. Records/habits/习惯名]]
- 当前连续: {N} 天
- 最长连续: {M} 天
- 建议: {打卡/更新记录}
```

---

## 主动关联流程

```
处理用户输入
    │
    ▼
提取关键实体 (人/地/事/时/情)
    │
    ▼
搜索已有记录
    │
    ├── 发现时间关联？ ──→ 添加到 related
    ├── 发现人物关联？ ──→ 添加到 related
    ├── 发现地点关联？ ──→ 添加到 related
    ├── 发现项目关联？ ──→ 添加到 related
    ├── 发现知识关联？ ──→ 添加到 related
    └── 发现情绪关联？ ──→ 添加到 related
    │
    ▼
组织输出
    │
    ├── 更新笔记的 related 字段
    ├── 添加关联提示到回复
    └── 发现模式时主动提醒
```


## 关联发现阈值

| 关联类型 | 触发阈值 | 说明 |
|---------|---------|------|
| 时间关联 | 同日/同期 | 必须关联 |
| 人物关联 | 出现 ≥2次 | 主动关联 |
| 地点关联 | 出现 ≥2次 | 主动关联 |
| 项目关联 | 明确相关 | 必须关联 |
| 知识关联 | 关键词匹配 | 主动关联 |
| 情绪关联 | 连续异常 | 提醒用户 |
| 习惯关联 | 行为匹配 | 建议打卡 |

---

## 关联维护

创建或更新笔记时：

1. **必须包含**: 时间关联 (同一文件内的不同部分)
2. **应该包含**: 项目关联、知识关联
3. **主动尝试**: 人物关联、地点关联
4. **条件触发**: 情绪关联、习惯关联

---

## 链接格式

使用 Obsidian 双链格式:
```markdown
[[笔记名]]           # 同仓库内链接
[[子目录/笔记名]]    # 带路径
[[笔记名#章节]]      # 链接到特定章节
[[笔记名|显示文本]]  # 带显示文本
```
